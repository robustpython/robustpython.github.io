Why Python

Thu 04 March 2021

Python is a great programming language to learn and use.

These are my reasons why I use it.

I also use other programming languages, but I think Python is best balanced language in terms of beauty, usefulness and practicality.

Syntax

Most mainstream languages today have C-style syntax (C, Java, JavaScript, C#, Go etc).

While this syntax is fine and familiar it is quite verbose and allows programmers to go really wild with the code formatting.

Python took a different approach and is not using curly braces to delimit code blocks. Rather it's using significant white spaces and code blocks are indented. Worth mentioning is the fact that in Python you don't use semicolon ; at the end of the line.

Take this function written in C-style language for example:

function my_function(p1, p2) {
    if (p1 > p2) {
        function_a();
      }
    else {
        function_b();
    }
}

you can also write it like

function my_function(p1, p2) {
    if (p1 > p2) {function_a();}
    else 
    { 
          function_b();
    function_c();
    }
}

and it will do exactly same thing.

We can write same function in Python like this:

def my_function(p1, p2):
    if p1 > p2:
        function_a()
    else:
        function_b()
        function_c()

But if you change it to

def my_function(p1, p2):
    if p1 > p2:
        function_a()
    else:
        function_b()
    function_c()

It is now a different function because function_c will always be called while before it was called only as a part of else branch.

This has an impact on readability and maintainability of the code as programmers are forced to style code well in order to make it run.

Python was designed to be highly readable language which is important because code is written once but read many times.

Python is also using english words instead of punctuations for logical operators. For AND operation it uses word and instead of &&, for OR it uses word or instead of || , for NOT it uses word not instead of !.

In C style language:

if (x > 5 && y > 8 || z == 10) {
    // Do the thing
}

In Python:

if x > 5 and y > 8 or z == 10:
    # Do the thing

Conciseness

This is partly related to syntax which we've discussed above but Python has some really great ways how to do things.

Python is very concise language.

It allows you to do a lot with very little code. That is important because it makes code easy and fast to read and write.

You don't have to write a lot of code that is not directly related to the thing you're trying to achieve. This saves you time and mental energy.

Let's say we want to check that variable number is bigger than 5 and smaller than 10. In C style languages you would do:

if (number > 5 && number < 10) {
    // Do the thing
}

in Python, we have chained comparison, and you can do:

if 5 < number < 10:
    # Do the thing

It's a similar story if you want to get part of a list (array). In Python, you can use slicing:

numbers = [1, 2, 3 , 4, 5]

first_two = numbers[:2]

while in C style languages you'd do:

let numbers = [1, 2, 3, 4, 5];

let first_two = numbers.slice(0, 2)

With in operator we can check if item is in the list or other data structure.

To check if number is in the list we'd write:

numbers = [1, 2, 3, 4, 5]

if 5 in numbers:
    # Do the thing

Or if a key is in the dictionary: 

if 'my_key' in my_dict:
   # Do the thing

It can be also used to check if a string contains another string 

if 'python' in 'Robust python':
    # Do the thing

Comprehensions are another great feature of Python which makes our code super concise.

You can create a list, a set or a dictionary in one line. You can also map and filter with them. For example, to create a list of squares of numbers from one to ten in C style language you'd do:

let squares = [];

for (i = 1; i <= 10; i++) { 
    squares.push(i*i)
}

In Python, we can achieve the same with:

squares = [i * i for i in range(1, 11)]

To get only squares bigger than 50, in Python we could just do:

big = [s for s in squares if s > 50]

This conciseness and clarity of Python makes you very productive as a developer while your code is still easily readable and understandable to others.

Multi paradigm

In Python, we can choose which programming paradigm we want to use for our program.

You can do most traditional imperative (procedural) programming where series of statements are executed in order.

Python also supports Object oriented programming paradigm where you create hierarchies of classes with methods. It supports polymorphism, inheritance (including multiple inheritance) and other OOP concepts.

Last but not the least Python has a great support for functional programming. It has list, dict, sets and generator expression - comprehensions.
All functions in Python are first class functions , so they can be passed around and manipulated as any other object. This allows it to have higher-order functions like map, filter and reduce. Lot of other functions to support functional programming is in itertools and functools modules.

Versatile

We can do pretty much anything in Python.

  • simple scripts to automate some trivial tasks
  • websites with millions of users
  • heavy data processing backends
  • APIs
  • desktop apps
  • software for IoT (Internet of Things, think RaspberryPi, Arduino etc.)
  • data visualisation
  • ML
  • and so much more

That is why many major companies (Amazon, Google, Netflix, Instagram, Dropbox, Spotify, NASA and so on) and millions of small companies and independent developer across the world use Python.

Runs almost everywhere

Python runs on most of the relevant platforms today. It runs on x86, x86_64 and ARM. It also runs on all major operating systems of today - Linux, Windows, macOS and even FreeBSD.

You can write your code where you prefer, and it can be run almost anywhere.

Community and libraries

Python is open source, independent and community driven language. It has a vast community of developers and enthusiasts.

They're very supportive and friendly which is huge benefit for both new and expert developers.

Community is dedicated to maintain and spread the language and that means there is an astonishing amount of libraries to help you with whatever you need to do.

Frameworks like Django and Flask (for web development), Numpy, Panda , NLTK, PyTorch (for ML and data science) and thousands of libraries for almost anything you can think of which are free to use and open source.

There are thousands of Python developers willing to help you with your problem if you just ask.

Machine Learning and Data science

Machine learning and data science are on the rise over the last few years and will continue to grow in coming years.

You can use any language for ML and data science related tasks but Python has become a preferred language for this domain. Most companies and even most of the tutorials and courses use Python as the language of choice for ML domain.

It's thanks to Python's strengths which we mentioned already above:

  • syntax and conciseness - which makes developers very productive and allows even beginners to write good code
  • exhaustive libraries which gives you solution to all existing problems
  • community build around whole ML domain

So, if you're interested in Machine learning then Python is definitely language you want to learn.

Gradual typing

Python is a dynamic language.

It has its advantages and disadvantages which I don't want to discuss here in detail, but it means you can write code without worrying about types and compiler which is checking them.

This is great if you need to just write a little script to automate something, or you want to build a prototype of the idea (Minimal Viable Product) or have a smaller project with few people.

However, for bigger projects with more developers types are very useful as they help you write more robust code, and you can catch more errors before the code is run.

It would look like this:

from typing import List

def function_a(a: int, b: List[str]):
    pass

In above function parameter a has to be of type int and parameter b must be list of strings - List[str].

Beauty of Python is that you can have both. You can write your code without types, but you can also add type annotations if you want to.

Then you can use static type checker like mypy to do the type checking for you to help you catch any possible errors. You can choose to use types from beginning of the project (or after you've build MVP), or you can start your project without types and introduce them much later when your code base and number of developers grow.

Career

Python has been around for 30 years and over time it's only growing in popularity. It is used in startups, small and medium companies as well as in big enterprises. This creates constant demand for Python programmers all over the world. And Python jobs come with high salaries. Whether you want to build a career as an employee in small or big company, or you want to be an independent developer or build your own business, Python is going to serve you well.
