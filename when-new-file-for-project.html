When to make a new .py file for a project?

Thu 29 April 2021

The problem

Most Python project are using multiple python files instead of having all code in one file.

When should you make a new python file for organization?

If you were making a big app, how would you determine that it would be best to make a new file?

Should you always try to separate code into multiple files?

Always split?

Let's start at the end. Should you always split code into multiple files?

No, I don't think so.

Many Python projects are just simple scripts that do one thing and are just few tens of lines of code. In this case there is not much point.

How to split?

The best way is to split your code into files based on WHAT it does.

Let's say your big app imports some data about companies (invoices, employees, customers), does some processing and then generates report.

You could start with following files: import.py, process.py, report.py.

When process.py file turns out to be a lot of code (approaching 1000 lines) it's time to split it further based on different parts of your processing e.g. process_invoices.py, process_employees.py and so on.

Then you'd probably start to organize them into folders - process folder for all the process files.

So split code logically based on what it does, don't allow files to get too big and group similar files with folder.

When should you split?

I said above that when file is a lot of code - approaching 1000 lines - it's time to split.

Why 1000? It's really just my observation and personal preference. How many lines is too many is mostly about personal preference.

Big files are hard to navigate. Scrolling through 5000 lines of code is not very practical.

Also when more than one person work on a project, and they use Git it will make things even harder if everything is in one or few very big files.

If you know from beginning that your project is not trivial always split logically based on what code does.

That should be your primary driver for splitting.

It is then much easier to navigate your project, find things and use version control (Git).
